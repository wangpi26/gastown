# 安全策略

## 报告漏洞

如果你在 Gas Town 中发现了安全漏洞，请负责任地报告：

1. **不要**为安全漏洞创建公开的 Issue
2. 直接通过邮件联系维护者并提供详细信息
3. 包含重现漏洞的步骤
4. 在公开披露前给予合理的修复时间

## 范围

Gas Town 是专注于多 Agent 协调的实验性软件。安全注意事项包括：

- **Agent 隔离**：Worker 在独立的 tmux 会话中运行，但共享文件系统访问权限
- **Git 操作**：Worker 可以推送到已配置的远程仓库
- **Shell 执行**：Agent 以当前运行用户的身份执行 shell 命令
- **Beads 数据**：工作追踪数据存储在 `.beads/` 目录中

## 最佳实践

使用 Gas Town 时：

- 在不受信任的代码环境中运行时使用隔离环境
- 在推送到生产分支前审查 Agent 输出
- 使用适当的 git 远程仓库权限
- 通过 `gt peek` 和日志监控 Agent 活动
